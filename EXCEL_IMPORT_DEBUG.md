# Excel 导入问题诊断指南

## 问题描述
部分 .xlsx 文件无法识别导入，需要诊断具体原因。

## 已添加的调试功能

### 1. 在 ViewModel 中添加了详细日志
位置：`ResidentViewModel.kt` 的 `importFromExcel` 方法

新增功能：
- 显示文件名和文件类型
- 显示导入结果统计（成功/失败）
- 捕获并显示异常信息
- 向用户显示更友好的错误提示

### 2. 在 ExcelImporter 中添加了详细日志
位置：`ExcelImporter.kt` 的 `importFromExcel` 和 `importXlsx` 方法

新增日志：
- 文件类型识别过程
- ZIP 包解析过程（条目数量、提取的文件）
- 共享字符串表解析（字符串数量、前5个示例）
- 表头解析结果
- 数据行解析（前3行的示例数据）
- 最终统计结果

## 如何使用调试日志

### 方法 1：通过 Android Studio Logcat 查看
1. 连接设备或启动模拟器
2. 在 Android Studio 中打开 Logcat
3. 过滤器设置：`ExcelImport`
4. 执行导入操作
5. 查看日志输出

### 方法 2：通过 adb 命令查看
```bash
adb logcat -s ExcelImport:D ExcelImport:W ExcelImport:E
```

## 常见问题诊断

### 问题 1：文件类型识别错误
**症状**：显示"未知文件类型"或尝试了多种格式都失败

**原因**：
- 文件扩展名不正确（如 .xls 实际是 .xlsx）
- 文件名被 URL 编码
- 文件损坏

**解决方法**：
- 检查日志中的"文件名"和"小写"字段
- 确认文件扩展名是否正确
- 尝试用 Excel 打开文件，确认文件可以正常打开

### 问题 2：表头不匹配
**症状**：显示"未找到「姓名」列"

**原因**：
- 表头名称不准确（如"名字"而不是"姓名"）
- 表头不在第一行
- 文件编码问题

**解决方法**：
- 查看日志中的"实际表头"
- 修改 Excel 文件的表头，确保与已知列名匹配
- 已知列名：姓名、性别、出生年月日、出生日期、年龄、受教育水平、学历、政治面貌、电话、手机、地址、住址

### 问题 3：ZIP 解析失败
**症状**：显示"无法读取工作表，文件可能已损坏"

**原因**：
- 文件不是真正的 .xlsx 文件
- 文件损坏
- 文件格式不支持

**解决方法**：
- 尝试用 Excel 打开文件并另存为 .xlsx
- 检查文件大小，如果是 0 字节说明文件为空
- 查看日志中的"ZIP 包共 X 个条目"，如果是 0 说明解析失败

### 问题 4：所有数据行姓名为空
**症状**：显示"没有找到可导入的数据"

**原因**：
- 姓名列索引错误
- 数据不在正确的列
- 姓名单元格为空或特殊格式

**解决方法**：
- 查看日志中的"第X行数据"示例
- 确认姓名列的数据是否正确读取
- 检查 Excel 文件中姓名列的数据是否正常

- 如果应用崩溃，查看崩溃日志
- 日志可能包含敏感数据（如居民姓名），请注意保护隐私
